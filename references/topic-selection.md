# 选题评估规则

> 3维度评分体系，用于评估选题的可行性、价值和差异化程度。

---

## 三维度评分模型

### 评分公式

```
选题总分 = 热度分 × 30% + 相关度分 × 40% + 切入价值分 × 30%
```

**及格线**：总分 ≥ 60分
**推荐线**：总分 ≥ 75分
**爆款线**：总分 ≥ 85分

---

### 维度一：热度（30%）

**评估内容**：选题当前的受关注程度。

| 评分项 | 分值 | 评估方法 |
|--------|------|----------|
| 搜索热度 | 0-25 | WebSearch搜索结果数量和质量 |
| 社交讨论度 | 0-25 | 微博/知乎/小红书讨论量 |
| 时效性 | 0-25 | 话题处于上升期=25，平稳期=15，衰退期=5 |
| 传播潜力 | 0-25 | 话题的"转发欲望"强度 |

**热度评分等级**：

| 等级 | 分值区间 | 特征 |
|------|----------|------|
| S级 | 90-100 | 全网热议，多平台热搜 |
| A级 | 75-89 | 行业内热议，部分出圈 |
| B级 | 60-74 | 目标受众关注，小范围讨论 |
| C级 | 40-59 | 少数人关注，缺乏讨论 |
| D级 | 0-39 | 几乎无人关注 |

**WebSearch热度查询模板**：

```
"{topic} 2025 热点 讨论"
"{topic} 趋势 数据 报告"
"{topic} 微博 知乎 热搜"
```

---

### 维度二：相关度（40%）

**评估内容**：选题与账号定位的匹配程度。

| 评分项 | 分值 | 评估方法 |
|--------|------|----------|
| 领域匹配 | 0-30 | 选题是否在账号核心领域内 |
| 人格匹配 | 0-25 | 选题是否适合当前写作人格 |
| 受众匹配 | 0-25 | 目标读者是否关心这个话题 |
| 能力匹配 | 0-20 | 是否有足够的知识/素材写好 |

**相关度评分等级**：

| 等级 | 分值区间 | 特征 |
|------|----------|------|
| 完美匹配 | 90-100 | 核心领域+擅长人格+核心受众 |
| 高度相关 | 75-89 | 核心领域或擅长人格 |
| 部分相关 | 60-74 | 边缘领域但受众有交集 |
| 低相关 | 40-59 | 勉强能写但缺乏优势 |
| 不相关 | 0-39 | 超出账号定位范围 |

**相关度快速判断**：

```yaml
relevance_check:
  domain_match:
    core: 30       # 核心领域（如账号定位是AI，选题是AI工具）
    adjacent: 20   # 相邻领域（如AI→科技）
    peripheral: 10 # 边缘领域（如AI→互联网）
    unrelated: 0   # 无关领域（如AI→美食）

  persona_match:
    perfect: 25    # 人格的最佳话题
    good: 18       # 人格可以驾驭
    acceptable: 12 # 人格能写但不是最佳
    poor: 5        # 人格不太适合

  audience_match:
    core: 25       # 核心受众直接关心
    potential: 18  # 核心受众可能关心
    marginal: 10   # 少数受众关心
    irrelevant: 0  # 受众不关心
```

---

### 维度三：切入价值（30%）

**评估内容**：账号能在该选题上提供的独特价值。

| 评分项 | 分值 | 评估方法 |
|--------|------|----------|
| 角度独特性 | 0-30 | 搜索前3页无相同角度 |
| 信息增量 | 0-25 | 能否提供别人没说的信息 |
| 可操作性 | 0-25 | 读者读后能否采取行动 |
| 持久价值 | 0-20 | 文章3个月后是否仍有价值 |

**切入价值评分等级**：

| 等级 | 分值区间 | 特征 |
|------|----------|------|
| 独创性切入 | 90-100 | 全新角度+独家信息+强行动 |
| 差异化切入 | 75-89 | 有独特角度或独家信息 |
| 有价值切入 | 60-74 | 有一定差异化 |
| 常规切入 | 40-59 | 与现有文章差异不大 |
| 无价值切入 | 0-39 | 完全重复现有内容 |

**切入价值自检问题**：

1. 搜索这个话题，前3篇文章说了什么？我能说不一样的话吗？
2. 我有什么别人没有的信息/经验/视角？
3. 读者看完能做什么之前做不到的事？
4. 3个月后这篇文章还有用吗？

## 历史去重规则

### 去重策略

| 策略 | 说明 | 触发条件 |
|------|------|----------|
| 严格去重 | 相同话题不重复写 | 30天内写过相同话题 |
| 角度去重 | 同一话题不同角度可写 | 超过30天且新角度与前次不同 |
| 系列去重 | 同一系列需间隔至少7天 | 同主题系列文章 |
| 常青更新 | 常青话题可90天后更新 | 数据/情况有重大变化 |

### 去重检查流程

```yaml
dedup_check:
  step_1:
    action: "搜索历史文章库"
    query: "topic keywords"
    time_range: "last_90_days"

  step_2:
    condition: "找到相似文章"
    actions:
      - within_30_days: "REJECT - 话题重复"
      - within_60_days:
          check: "角度是否不同"
          if_different: "ACCEPT with 新角度标记"
          if_same: "REJECT - 角度重复"
      - within_90_days:
          check: "是否有重大新信息"
          if_yes: "ACCEPT as 更新型"
          if_no: "REJECT - 价值不足"

  step_3:
    condition: "未找到相似文章"
    action: "ACCEPT - 新话题"
```

## 常青选题

### 定义

常青选题是指不受时间限制、长期有搜索需求的话题。

### 常青选题识别标准

| 标准 | 说明 |
|------|------|
| 搜索稳定性 | 过去12个月搜索量波动<30% |
| 长期需求 | 3年内预计仍有需求 |
| 知识型内容 | 方法论/技能/原理类 |
| 非时效性 | 不依赖特定事件/时间点 |

### 常青选题库

| 领域 | 常青选题 | 更新周期 |
|------|----------|----------|
| 职场 | 时间管理方法、沟通技巧、职业规划 | 6个月 |
| AI/科技 | AI工具盘点、入门教程、概念科普 | 3个月 |
| 财务 | 理财入门、消费观、副业方法 | 6个月 |
| 成长 | 读书方法、学习策略、思维模型 | 12个月 |
| 管理 | 团队管理、决策方法、目标设定 | 12个月 |

### 常青选题使用规则

1. 常青选题优先级低于热点选题，但可作"填充选题"
2. 每月至少1篇常青选题文章
3. 常青选题更新时必须在开头注明更新信息
4. 常青选题的SEO权重高于热点选题（长期流量）

## 选题评估完整流程

```yaml
topic_evaluation_pipeline:
  step_1_propose:
    action: "提出候选选题（1-3个）"
    source: "热点/常青库/用户需求"

  step_2_heat_score:
    action: "WebSearch评估热度"
    queries:
      - "{topic} 热点 趋势 2025"
      - "{topic} 讨论 争议"
    output: "热度分（0-100）"

  step_3_relevance_score:
    action: "对照账号配置评估相关度"
    checks:
      - domain_match
      - persona_match
      - audience_match
      - capability_match
    output: "相关度分（0-100）"

  step_4_angle_score:
    action: "评估切入价值"
    checks:
      - uniqueness (WebSearch前3页)
      - information_delta
      - actionability
      - evergreen_value
    output: "切入价值分（0-100）"

  step_5_dedup:
    action: "历史去重检查"
    output: "通过/拒绝/需调整角度"

  step_6_final_score:
    formula: "热度×30% + 相关度×40% + 切入价值×30%"
    output: "总分（0-100）"
    decision:
      - "≥85: 立即写"
      - "75-84: 推荐写"
      - "60-74: 可以写，需优化角度"
      - "<60: 暂缓，换选题"

  step_7_plan:
    action: "确定框架、人格、增强策略"
    output: "选题执行计划"
```
