# 合规/SEO参考

> 微信合规红线、低创检测对策、CSS随机化范围、图片标注规则。

## 一、微信合规红线

### 1.1 绝对红线（触碰即封）

| 红线 | 说明 | 应对 |
|------|------|------|
| 政治敏感内容 | 涉及政治人物、政策评价、政治体制 | 选题阶段即规避，不碰不试 |
| 色情/低俗 | 裸露、暗示、性相关内容 | 不生成、不引用 |
| 暴力/恐怖 | 血腥、暴力、恐吓内容 | 不生成、不引用 |
| 赌博/诈骗 | 赌博方法、诈骗教程 | 不生成、不引用 |
| 传销/非法集资 | 传销模式、非法集资 | 不推荐任何"快速致富"模式 |
| 药品/医疗器械 | 未经审批的药品/器械推荐 | 不做具体药品推荐 |
| 侵犯隐私 | 个人隐私信息 | 不引用真实个人隐私数据 |
| 伪造信息 | 假新闻、伪造数据 | 所有数据必须有来源 |

### 1.2 高危区域（容易踩线）

| 区域 | 风险 | 应对策略 |
|------|------|----------|
| 金融建议 | 可能被视为投资建议 | 加免责声明，用"参考"而非"建议" |
| 健康建议 | 可能被视为医疗建议 | 加免责声明，不代替就医 |
| 教育培训 | 夸大宣传风险 | 不承诺具体效果 |
| 职场建议 | 可能引发争议 | 标注"仅供参考" |
| 法律相关 | 可能被视为法律意见 | 加免责声明，建议咨询专业人士 |
| 未成年人相关 | 未成年人保护法规 | 不针对未成年人内容 |

### 1.3 合规检查清单

```yaml
compliance_check:
  content:
    - "无政治敏感内容"
    - "无色情/低俗内容"
    - "无暴力/恐怖内容"
    - "无赌博/诈骗内容"
    - "无传销/非法集资内容"
    - "无药品/医疗器械推荐"
    - "无个人隐私信息"
    - "无伪造数据/假新闻"

  financial_content:
    - "不承诺投资收益"
    - "不推荐具体股票/基金代码"
    - "有免责声明"
    - "用'参考'而非'建议'"

  health_content:
    - "不替代医疗建议"
    - "有免责声明"
    - "不推荐具体药品"

  disclaimer_required:
    - "涉及投资/理财→加'投资有风险，仅供参考'"
    - "涉及健康→加'不替代专业医疗建议'"
    - "涉及法律→加'不构成法律意见'"
```

## 二、低创检测对策

### 2.1 低创检测机制

微信对"低创内容"的检测维度：

| 检测维度 | 检测内容 | 低创信号 |
|----------|----------|----------|
| 原创性 | 与已有文章的相似度 | 相似度>60% |
| 信息密度 | 有效信息/总字数 | 密度<30% |
| 结构复杂度 | 段落/句式变化 | 结构过于简单 |
| 搬运痕迹 | 排版/格式与其他文章一致 | 格式完全一致 |
| AI痕迹 | 典型AI写作模式 | 大量AI标记词 |
| 互动质量 | 读者停留和互动 | 完读率极低 |

### 2.2 对策：原创性保障

| 对策 | 说明 | 执行要点 |
|------|------|----------|
| 角度唯一 | 每篇文章有独特切入角度 | angle_discovery策略 |
| 信息增量 | 提供别人没有的信息 | density_boost策略 |
| 个人经验 | 注入真实个人经历 | real_feel策略 |
| 数据支撑 | 用独特数据佐证 | WebSearch查证 |

### 2.3 对策：反AI检测

| 对策 | 说明 | 参考 |
|------|------|------|
| 统计层去AI | 句长方差、词汇温度等 | writing-guide.md 第一层 |
| 语言层去AI | 禁用词、碎句、意外词 | writing-guide.md 第二层 |
| 内容层去AI | 信息锚定、具体化 | writing-guide.md 第三层 |
| 实时自检 | 每500字5项检查 | realtime-check.md |

### 2.4 对策：低密度防范

| 对策 | 说明 | 执行要点 |
|------|------|----------|
| 信息密度波 | 密度有高有低，不是均匀低密度 | density_wave_enabled: true |
| 去除水话 | 删除不增加信息的句子 | 副词删除、空洞句压缩 |
| 数据填充 | 用具体数字替代模糊描述 | info_anchor_density: ≥2 |
| 案例充实 | 每个论点配具体案例 | 清单项≥1个案例 |

### 2.5 低创检测评分

```yaml
low_creation_check:
  scoring:
    originality: 30         # 原创性
    info_density: 25        # 信息密度
    structure_complexity: 20 # 结构复杂度
    ai_trace: 15            # AI痕迹（负分）
    engagement: 10          # 互动预期

  thresholds:
    pass: 60                # ≥60分通过
    warning: 40-59          # 需优化
    fail: <40               # 不可发布

  auto_fix:
    - "ai_trace > 10分：执行反AI检测修复"
    - "info_density < 20分：执行density_boost"
    - "structure_complexity < 15分：调整段落结构"
```

## 三、CSS随机化范围

### 3.1 为什么需要CSS随机化

微信可能通过CSS指纹识别AI批量生成的文章。对样式参数进行微随机化可以降低批量检测风险。

### 3.2 随机化参数范围

| CSS属性 | 标准值 | 随机范围 | 单位 |
|---------|--------|----------|------|
| `font-size`（正文） | 15 | 14.5-15.5 | px |
| `font-size`（小标题） | 17 | 16.5-17.5 | px |
| `line-height` | 1.75 | 1.7-1.85 | 数值 |
| `margin-bottom`（段落） | 14 | 12-16 | px |
| `padding`（卡片） | 16 | 14-18 | px |
| `border-radius`（卡片） | 8 | 6-10 | px |
| `letter-spacing` | 0.5 | 0.3-0.7 | px |
| `color`（正文） | #333333 | #2d2d2d-#3a3a3a | 十六进制微调 |
| `border-left-width`（引用） | 3 | 3-4 | px |

### 3.3 随机化规则

```yaml
css_randomization:
  enabled: true
  strategy: "per_article"     # 每篇文章一套随机值
  seed_source: "article_id"   # 用文章ID作为随机种子（同文章一致）

  rules:
    - "同篇文章内样式参数一致"
    - "不同文章间样式参数有微小差异"
    - "差异不可被人眼察觉（±5%以内）"
    - "不影响阅读体验"

  apply_to:
    - "段落间距"
    - "行高"
    - "字体大小"
    - "圆角大小"
    - "内边距"
    - "字间距"

  do_not_apply_to:
    - "主色调（品牌一致性）"
    - "强调色（品牌一致性）"
    - "代码块样式（需统一）"
    - "图片尺寸（需统一）"
```

## 四、图片标注规则

### 4.1 标注类型

| 标注类型 | 何时使用 | 标注内容 |
|----------|----------|----------|
| AI生成标注 | AI生成的图片 | "AI生成" |
| 来源标注 | 引用的第三方图片 | "来源：{来源名}" |
| 数据来源 | 数据图表 | "数据来源：{来源名}" |
| 免责标注 | 截图/引用 | "图片仅供参考" |

### 4.2 标注位置和样式

```yaml
image_attribution:
  position: "图片下方居中"
  style:
    font_size: "12px"
    color: "#999999"
    text_align: "center"
    margin_top: "4px"

  templates:
    ai_generated: "图片由AI生成"
    source: "图片来源：{source_name}"
    data_source: "数据来源：{source_name}，{year}"
    screenshot: "截图时间：{date}"
```

### 4.3 标注规则

| 规则 | 说明 |
|------|------|
| AI生成必标 | 所有AI生成的图片必须标注"AI生成" |
| 来源可溯 | 引用图片必须标注出处 |
| 数据可查 | 数据图表必须标注数据来源 |
| 不误导 | 标注不能暗示AI图片为真实照片 |
| 合规要求 | 部分地区法律要求AI内容标注 |

### 4.4 图片合规检查

```yaml
image_compliance_check:
  - "AI生成图片已标注"
  - "引用图片有出处"
  - "数据图表有来源"
  - "无水印残留"
  - "无版权争议"
  - "无肖像权问题"
  - "无敏感内容"
```

## 五、完整合规/SEO检查流程

```yaml
compliance_seo_pipeline:
  step_1_content_compliance:
    check: "合规红线检查"
    action: "自动检查所有红线项"
    on_fail: "REJECT - 不可发布"

  step_2_low_creation_check:
    check: "低创检测评分"
    action: "计算低创评分"
    on_fail:
      - "<40: REJECT"
      - "40-59: WARNING + auto_fix"

  step_3_ai_trace_check:
    check: "AI痕迹检测"
    action: "统计AI标记词、句式模式"
    on_fail: "执行反AI修复"

  step_4_seo_check:
    check: "SEO规范检查"
    action: "标题/摘要/关键词/标签"
    on_fail: "WARNING + 建议"

  step_5_css_randomization:
    check: "CSS随机化"
    action: "为本篇文章生成随机CSS参数"
    on_fail: "使用默认值"

  step_6_image_attribution:
    check: "图片标注"
    action: "检查所有图片的标注"
    on_fail: "WARNING + 自动补充"

  step_7_final_check:
    check: "最终确认"
    action: "所有检查通过后发布"
    on_fail: "列出问题清单，等待修复"
```
