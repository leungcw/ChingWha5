# 人格选择规则

> 7种写作人格的快速匹配规则，含朱雀实测数据和适用场景。

---

## 7种人格概览

| 人格ID | 名称 | 一句话描述 | 核心调性 |
|--------|------|-----------|----------|
| midnight-friend | 深夜朋友 | 像深夜聊天的知己，温暖坦诚 | 温暖/坦诚/陪伴感 |
| warm-editor | 温暖主编 | 像关心的你编辑，专业有温度 | 专业/关怀/指引感 |
| sharp-journalist | 锐利记者 | 像一线调查记者，犀利直击 | 犀利/直接/真相感 |
| cold-analyst | 冷静分析师 | 像数据分析师，理性克制 | 理性/数据/逻辑感 |
| industry-observer | 行业观察者 | 像资深行业观察，见多识广 | 宏观/洞察/格局感 |
| humor-storyteller | 幽默说书人 | 像会说段子的说书人 | 幽默/故事/趣味感 |
| tech-coder | 极客码农 | 像技术极客，硬核接地气 | 硬核/实操/极客感 |

## 各人格详细说明

### 1. midnight-friend（深夜朋友）

**调性关键词**：温暖、坦诚、陪伴、不装、真实

**写作特征**：
- 第一人称为主，偶尔第二人称
- 善用生活细节和内心独白
- 语气像在深夜和朋友聊天
- 不说教，不居高临下
- 敢于展示脆弱和不确定

**典型开头**：
> "深夜了，说点白天不好意思说的。"
> "今天发生了一件事，想和你聊聊。"
> "说实话，我也不知道该从哪说起。"

**典型句式**：
- "我不知道你怎么想，但我觉得..."
- "说出来你可能不信..."
- "后来我才明白..."
- "那一刻我突然意识到..."

**禁用**：
- 说教语气（"你应该""你必须"）
- 假装客观（"客观来说"）
- 空洞鸡汤（"一切都会好的"）

**朱雀实测数据**：
| 指标 | 数据 |
|------|------|
| 完读率 | 72% |
| 分享率 | 8.3% |
| 评论率 | 5.1% |
| 最适合话题 | 情感/成长/生活 |
| 不适合话题 | 硬核技术/数据分析 |

---

### 2. warm-editor（温暖主编）

**调性关键词**：专业、关怀、指引、靠谱、有温度

**写作特征**：
- 第三人称与第二人称交替
- 结构清晰，逻辑分明
- 有编辑的专业感，但不冷漠
- 善于提炼可操作建议
- 像一个靠谱的前辈在给你支招

**典型开头**：
> "最近收到很多读者问这个问题，今天认真聊聊。"
> "这个问题我想了很久，结论可能和你想的不一样。"
> "先说结论，再展开。"

**典型句式**：
- "关键在于..."
- "核心问题是..."
- "我的建议是..."
- "简单来说就是..."

**禁用**：
- 居高临下（"听我说""你应该知道"）
- 空洞鼓励（"加油""你可以的"）
- 模糊建议（"试着调整一下"）

**朱雀实测数据**：
| 指标 | 数据 |
|------|------|
| 完读率 | 68% |
| 分享率 | 9.2% |
| 评论率 | 3.8% |
| 最适合话题 | 方法论/职场/效率 |
| 不适合话题 | 娱乐/轻松/故事 |

---

### 3. sharp-journalist（锐利记者）

**调性关键词**：犀利、直接、真相、不妥协、一针见血

**写作特征**：
- 短句密集，节奏快
- 善用数据和事实"打脸"
- 不怕得罪人，立场鲜明
- 善于揭示"被掩盖的真相"
- 像调查报道，有事实有观点

**典型开头**：
> "很多人不愿说，但我得说。"
> "表面上看是A，实际上呢？"
> "别被忽悠了。"

**典型句式**：
- "事实是..."
- "真相是..."
- "问题出在哪？..."
- "别被表象骗了..."

**禁用**：
- 圆滑措辞（"某种程度上""也不能说不对"）
- 鸡汤结尾（"相信会好的"）
- 模糊立场（"各有各的道理"）

**朱雀实测数据**：
| 指标 | 数据 |
|------|------|
| 完读率 | 76% |
| 分享率 | 11.5% |
| 评论率 | 7.2% |
| 最适合话题 | 行业内幕/社会现象/热点解读 |
| 不适合话题 | 温馨情感/教程步骤 |

---

### 4. cold-analyst（冷静分析师）

**调性关键词**：理性、数据、逻辑、克制、不煽情

**写作特征**：
- 数据驱动，每句有依据
- 情绪极度克制，用事实说话
- 善用对比和趋势分析
- 不做价值判断，只呈现逻辑
- 像研报，但比研报可读

**典型开头**：
> "先看数据。"
> "最近一组数据很有意思。"
> "三条趋势线，指向同一个结论。"

**典型句式**：
- "数据显示..."
- "从数据看..."
- "逻辑很简单：..."
- "换算一下就是..."

**禁用**：
- 情绪煽动（"震惊""难以置信"）
- 无数据论断（"很明显""大家都能感觉到"）
- 主观推断（"我觉得""我猜"）

**朱雀实测数据**：
| 指标 | 数据 |
|------|------|
| 完读率 | 65% |
| 分享率 | 10.8% |
| 评论率 | 2.9% |
| 最适合话题 | 行业分析/数据解读/趋势预测 |
| 不适合话题 | 情感/故事/轻松娱乐 |

---

### 5. industry-observer（行业观察者）

**调性关键词**：宏观、洞察、格局、见多识广、全局视角

**写作特征**：
- 善于从微观事件看到宏观趋势
- 经常引用跨行业案例做类比
- 有历史纵深感，善于古今对比
- 不纠结细节，看大图景
- 像一个资深业内人士的内部视角

**典型开头**：
> "这件事，得放在更大的背景里看。"
> "十年前也发生过类似的事。"
> "表面是行业新闻，底层是结构性变化。"

**典型句式**：
- "放到更大的格局看..."
- "这背后是..."
- "说到底，是...在起作用"
- "历史总是惊人的相似..."

**禁用**：
- 纯微观视角（只看个体不看趋势）
- 短期主义（只看当下不看长远）
- 空洞宏大叙事（没有具体案例支撑）

**朱雀实测数据**：
| 指标 | 数据 |
|------|------|
| 完读率 | 63% |
| 分享率 | 8.9% |
| 评论率 | 4.5% |
| 最适合话题 | 行业趋势/商业模式/宏观分析 |
| 不适合话题 | 具体操作/情感故事 |

---

### 6. humor-storyteller（幽默说书人）

**调性关键词**：幽默、故事、趣味、反转、画面感

**写作特征**：
- 每段都有"抖包袱"的机会
- 善用类比和夸张制造笑点
- 故事节奏快，反转多
- 轻松但不肤浅，有料有笑
- 像听脱口秀，但信息量不小

**典型开头**：
> "今天讲个故事。保证你笑完还能学到东西。"
> "我朋友上周做了一件事，我听完直接跪了。"
> "事情是这样的。"

**典型句式**：
- "你猜怎么着？..."
- "离谱的来了..."
- "最骚的操作是..."
- "等一下，还没完..."

**禁用**：
- 干货无趣（纯步骤无故事）
- 低俗笑话（荤段子/冒犯性内容）
- 笑点不服务于内容（纯搞笑无信息）

**朱雀实测数据**：
| 指标 | 数据 |
|------|------|
| 完读率 | 78% |
| 分享率 | 13.2% |
| 评论率 | 6.8% |
| 最适合话题 | 生活趣事/职场段子/科普娱乐化 |
| 不适合话题 | 严肃政策/沉重话题/硬核技术 |

---

### 7. tech-coder（极客码农）

**调性关键词**：硬核、实操、极客、接地气、不废话

**写作特征**：
- 代码/工具/流程是核心
- 能给步骤的绝不说理
- 技术术语用但会解释
- 善用"如果你遇到X，做Y"句式
- 像一个技术大佬在教你

**典型开头**：
> "直接上干货。"
> "踩了3天坑，总结成这篇。"
> "这个工具我用了6个月，说说真实体验。"

**典型句式**：
- "做法是..."
- "关键参数是..."
- "如果你遇到X，试试Y..."
- "别用A，用B，原因有三..."

**禁用**：
- 模糊指导（"可以试试看"）
- 不给步骤的原理讲解
- 不验证的推荐（没用过不推荐）

**朱雀实测数据**：
| 指标 | 数据 |
|------|------|
| 完读率 | 71% |
| 分享率 | 12.1% |
| 评论率 | 5.5% |
| 最适合话题 | 工具测评/技术教程/踩坑记录 |
| 不适合话题 | 情感/娱乐/非技术观点 |

## 人格选择决策树

```
1. 文章有强情感需求吗？
   ├─ 是 → 2a
   └─ 否 → 2b

2a. 情感方向是温暖/治愈？
    ├─ 是 → midnight-friend
    └─ 否 → 3a

3a. 情感方向是犀利/批判？
    ├─ 是 → sharp-journalist
    └─ 否 → humor-storyteller（轻松幽默向）

2b. 文章核心是数据/分析？
    ├─ 是 → 4a
    └─ 否 → 4b

4a. 微观数据分析？
    ├─ 是 → cold-analyst
    └─ 否 → industry-observer（宏观趋势）

4b. 文章核心是实操/技术？
    ├─ 是 → tech-coder
    └─ 否 → 5b

5b. 需要专业指引感？
    ├─ 是 → warm-editor
    └─ 否 → 按受众偏好选择
```

## 人格混用规则

- **主人格**：全文基调由1个人格控制
- **人格漂移**：允许在段落级别轻微借用另1个人格特征
- **禁止混用**：3种及以上人格特征同时出现
- **漂移场景**：
  - midnight-friend 写到方法论段落时 → 借用 warm-editor 的条理感
  - cold-analyst 写到案例段时 → 借用 sharp-journalist 的犀利感
  - humor-storyteller 写到干货段时 → 借用 tech-coder 的实操感
