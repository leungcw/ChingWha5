# 内容增强策略

> 4种内容增强策略，在写作过程中动态提升文章的信息密度、角度独特性、细节真实感和情感共鸣。

---

## 策略一：angle_discovery（角度发现）

**核心逻辑**：在常见话题中找到被忽略的独特角度，让文章从"又一个XX文章"变成"唯一从XX角度看的文章"。

### 适用框架映射

| 框架 | 角度发现方式 |
|------|-------------|
| A 痛点型 | 痛点归因的反常识角度 |
| B 故事型 | 故事的"另一面"视角 |
| C 清单型 | 清单的逆向排序逻辑 |
| D 对比型 | 第三选择（非A非B） |
| E 热点解读 | 被主流叙事忽略的维度 |
| F 纯观点 | 支持反对者的理由 |
| G 复盘/体验 | "如果重来"的假设视角 |

### WebSearch 查询模板

```
# 通用角度发现查询
"{topic} 反常识 被忽略 角度"
"{topic} 另一面 不同观点"
"{topic} 争议 辩论 核心分歧"

# 行业话题
"{topic} 行业内幕 从业者 真实看法"
"{topic} 失败案例 反面教材"

# 社会现象
"{topic} 利益链 谁获益"
"{topic} 历史对比 过去怎样"
```

### 执行步骤

1. **主题解构**：将话题拆解为3-5个核心维度
2. **主流角度识别**：列出已有文章的3个常见角度
3. **角度搜索**：使用WebSearch查询模板，收集非常规视角
4. **角度评估**：按以下标准评估每个候选角度
   - 新颖度（0-10）：该角度是否被充分讨论过？
   - 可信度（0-10）：是否有事实支撑？
   - 共鸣度（0-10）：读者是否关心？
   - 可写性（0-10）：素材是否足够？
5. **角度确认**：选择综合分最高的角度，或组合2个角度
6. **角度锚定**：在开头第2段明确点出角度

### 验证标准

| 检查项 | 标准 | 不通过处理 |
|--------|------|-----------|
| 角度独特性 | 前3页搜索结果无相同角度 | 换角度或深化 |
| 角度可信度 | 有≥2个可引用的来源 | WebSearch补充 |
| 角度一致性 | 全文角度统一无偏移 | 调整段落结构 |
| 角度价值 | 能回答"所以呢？" | 深化到可行动层面 |

## 策略二：density_boost（密度提升）

**核心逻辑**：在不增加字数的前提下，通过信息压缩和论据丰富化提升信息密度，让每句话都"值得读"。

### 适用框架映射

| 框架 | 密度提升重点 |
|------|-------------|
| A 痛点型 | 原因剖析段的数据密度 |
| B 故事型 | 转折点的信息密度 |
| C 清单型 | 每个清单项的论据密度 |
| D 对比型 | 深层原因的分析密度 |
| E 热点解读 | 真相揭示段的信息密度 |
| F 纯观点 | 核心论证的论据密度 |
| G 复盘/体验 | 执行过程和结果的数据密度 |

### WebSearch 查询模板

```
# 数据补充查询
"{topic} 数据 统计 2024 2025"
"{topic} 报告 调研 白皮书"
"{topic} 市场规模 增长率"

# 案例补充查询
"{topic} 案例 成功/失败 真实"
"{topic} 对比 A vs B 数据"

# 权威引用查询
"{topic} 专家 观点 研究 论文"
"{topic} 官方 公告 政策 规定"
```

### 执行步骤

1. **密度检测**：对当前草稿进行信息密度扫描
   - 标记"空洞段"：无数据、无案例、无具体信息的段落
   - 标记"重复段"：同一信息用不同措辞重复表达
   - 标记"冗余段"：可以压缩但不影响理解的段落
2. **空洞段填充**：
   - 优先补充数据锚点（具体数字+来源）
   - 其次补充案例锚点（具体人物+事件+结果）
   - 最后补充逻辑锚点（因果链、对比、类比）
3. **重复段合并**：将相同信息合并，腾出空间给新信息
4. **冗余段压缩**：用1句话替代2句话，用1个词替代1个短语
5. **密度校验**：重新扫描，确保无空洞段

### 验证标准

| 检查项 | 标准 | 不通过处理 |
|--------|------|-----------|
| 空洞段率 | ≤5% | 继续填充 |
| 数据密度 | 每500字≥1个具体数字 | WebSearch补充 |
| 案例密度 | 每篇≥2个具体案例 | 补充案例 |
| 重复率 | 同一信息不重复表达 | 合并压缩 |
| 信息/字数比 | 信息密度波目标达成 | 调整密度分布 |

## 策略三：detail_anchoring（细节锚定）

**核心逻辑**：用具体、可感的细节替代抽象描述，让读者"看到"而不是"被告知"。

### 适用框架映射

| 框架 | 细节锚定重点 |
|------|-------------|
| A 痛点型 | 痛点场景的感官细节 |
| B 故事型 | 关键场景的五感细节 |
| C 清单型 | 每项的具体执行细节 |
| D 对比型 | A/B两面的生活细节 |
| E 热点解读 | 事件现场的具体细节 |
| F 纯观点 | 论据中的真实案例细节 |
| G 复盘/体验 | 执行过程中的真实细节 |

### WebSearch 查询模板

```
# 场景细节查询
"{topic} 具体经历 个人故事 细节"
"{topic} 一天 日常 真实生活"

# 数据细节查询
"{topic} 具体数字 精确数据 明细"
"{topic} 地区/公司/人群 分布 差异"

# 体验细节查询
"{topic} 使用体验 真实感受 优缺点"
"{topic} 对比 实测 亲测"
```

### 执行步骤

1. **抽象标记**：扫描草稿，标记所有抽象表述
   - 类型A：模糊量词（很多、大部分、显著）
   - 类型B：泛指名词（大家、企业、社会）
   - 类型C：空洞形容词（重要的、优秀的、出色的）
   - 类型D：无场景陈述（直接给结论无铺垫）
2. **替换策略**：
   - 类型A → 具体数字（"很多"→"超过80%"）
   - 类型B → 具体对象（"企业"→"年营收5000万以上的中小企业"）
   - 类型C → 具体描述（"重要的"→"错过就损失3个月的"）
   - 类型D → 场景铺设（先描述1-2句场景，再给结论）
3. **五感注入**：在关键场景补充感官细节
   - 视觉：颜色、光线、空间
   - 听觉：声音、音量、节奏
   - 触觉：温度、质感、力度
   - 嗅觉：气味、清新/浑浊
   - 味觉：苦/甜/酸（比喻性也可）
4. **细节校验**：确保每个细节服务于论点，不为细节而细节

### 验证标准

| 检查项 | 标准 | 不通过处理 |
|--------|------|-----------|
| 抽象表述率 | ≤20% | 继续替换 |
| 五感覆盖 | 关键场景≥3种感官 | 补充感官细节 |
| 细节相关度 | 每个细节与论点相关 | 删除无关细节 |
| 细节可信度 | 细节可验证或不矛盾 | 修正或删除 |
| 信息锚点数 | 每500字≥2个 | 补充锚点 |

## 策略四：real_feel（真实感受注入）

**核心逻辑**：注入真实的情感体验和个人化表达，打破AI写作的"正确但无感"特征。

### 适用框架映射

| 框架 | 真实感受注入方式 |
|------|-----------------|
| A 痛点型 | 痛点的个人体验描述 |
| B 故事型 | 内心独白和情绪波动 |
| C 清单型 | 个人踩坑/受益的亲历 |
| D 对比型 | 选择时的纠结和判断 |
| E 热点解读 | 第一反应和情绪变化 |
| F 纯观点 | 立场形成的个人经历 |
| G 复盘/体验 | 过程中的真实感受 |

### WebSearch 查询模板

```
# 真实体验查询
"{topic} 个人经历 真实感受 踩坑"
"{topic} 后悔 经验教训 如果重来"

# 情感共鸣查询
"{topic} 共鸣 扎心 说到心坎"
"{topic} 焦虑 迷茫 困惑 真实"

# 情绪表达查询
"{topic} 吐槽 真话 大实话"
"{topic} 职场/生活 真相 残酷"
```

### 执行步骤

1. **情感标注**：对草稿进行情感浓度扫描
   - 标记"冷漠段"：纯客观陈述，无情感色彩
   - 标记"假情段"：用"不禁感叹""由衷地"等AI情感词
   - 标记"真情机会"：可以注入真实感受的位置
2. **冷漠段升温**：
   - 加入第一人称感受（"我当时的第一反应是..."）
   - 加入具体情绪词（不是"感慨"，是"心凉了半截"）
   - 加入内心疑问（"我当时就想，这合理吗？"）
3. **假情段替换**：
   - 删除"不禁感叹"类AI情感标记词
   - 用具体描述替代抽象情感（"感动"→"鼻子一酸"）
   - 用行动替代形容（"我很担心"→"那晚我翻来覆去睡不着"）
4. **真实感增强技巧**：
   - **坦诚式**：承认自己的局限（"说实话，我也不确定..."）
   - **反问式**：提出没答案的问题（"这到底对不对？我不知道。"）
   - **矛盾式**：展示内心冲突（"理智告诉我要坚持，但身体很诚实。"）
   - **细节式**：用微小动作传递情绪（"手指在发送键上停了3秒。"）
5. **感受校验**：确保注入的感受与人格设定一致

### 验证标准

| 检查项 | 标准 | 不通过处理 |
|--------|------|-----------|
| 冷漠段率 | ≤15% | 注入情感 |
| 假情词率 | 0个 | 全部替换 |
| 情感一致性 | 与人格设定匹配 | 调整表达方式 |
| 情感真实性 | 读起来不像"正确的空话" | 用具体细节替代 |
| 情绪曲线 | 有波动，非单调 | 调整情绪节奏 |

---

## 策略组合建议

| 文章类型 | 推荐策略组合 | 优先级 |
|----------|-------------|--------|
| 深度分析 | angle_discovery + density_boost | 先角度后密度 |
| 情感故事 | detail_anchoring + real_feel | 先细节后感受 |
| 热点解读 | angle_discovery + density_boost | 先角度后密度 |
| 方法分享 | density_boost + detail_anchoring | 先密度后细节 |
| 个人经验 | detail_anchoring + real_feel | 先细节后感受 |
| 观点输出 | angle_discovery + real_feel | 先角度后感受 |
| 全面增强 | 全4策略 | 角度→密度→细节→感受 |

### 增强执行顺序

```
1. angle_discovery（写作前，确定角度）
2. density_boost（写作中，填充信息）
3. detail_anchoring（修改中，替换抽象）
4. real_feel（润色中，注入情感）
```
