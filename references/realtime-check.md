# 写作中实时自检

> 写作过程中每500字自动执行5项检查，确保反AI写作规范持续生效。

---

## 自检触发规则

| 规则 | 说明 |
|------|------|
| 触发间隔 | 每写完500字触发一次 |
| 触发方式 | 自动触发，无需手动调用 |
| 检查范围 | 最近500字 |
| 检查耗时 | 每项检查 ≤3秒 |
| 不通过处理 | 标记问题，写完当前段落后修正 |

---

## 五项检查

### 检查1：句长交替（Sentence Length Alternation）

**检查目标**：确保近期文本的句长有足够波动，避免AI式的均匀句长。

**检查方法**：
1. 提取最近500字的所有句子（以句号、问号、感叹号分隔）
2. 计算每句字数
3. 检查以下指标：

| 指标 | 通过标准 | 不通过处理 |
|------|----------|-----------|
| 句长标准差 | ≥8字 | 插入1个极短句（≤8字）或拆分1个长句 |
| 最短句 | ≤10字 | 缩短某句或插入碎句 |
| 最长句 | ≥30字 | 如无长句，合并2个相关短句 |
| 连续句长差 | 连续3句长度差<5字≤1次 | 调整中间句的长度 |
| 短长比 | 短句(≤15字)占比20-40% | 增加或减少短句 |

**快速修正动作**：
- 动作A：将某个长句末尾拆为独立碎句
  - 例："这种做法在大多数情况下是合理的" → "这种做法在大多数情况下是合理的。至少目前是这样。"
- 动作B：用短句替换长句中的从句
  - 例："虽然很多人认为这样做是对的，但我觉得有问题" → "很多人觉得对。但我不这么看。"

---

### 检查2：情绪锚定（Emotional Anchoring）

**检查目标**：确保近期文本有明确的情绪方向，不是"温水"状态。

**检查方法**：
1. 扫描最近500字，标注每段的情绪极性
2. 检查以下指标：

| 指标 | 通过标准 | 不通过处理 |
|------|----------|-----------|
| 情绪极性分布 | 至少覆盖2种极性 | 插入1句反极性表述 |
| 情绪强度 | 至少1段有明显情绪 | 加入情绪词或感叹 |
| 极性单调 | 无连续3段同极性 | 在第3段插入转折 |
| 全局曲线 | 近500字情绪在整体曲线上的位置合理 | 调整情绪强度 |

**情绪极性速查**：

| 极性 | 信号词 | 强度 |
|------|--------|------|
| 强正 | 终于！/太牛了/绝了 | ★★★ |
| 弱正 | 其实还不错/挺好的 | ★☆☆ |
| 中性 | 也就是说/具体来看 | ☆☆☆ |
| 弱负 | 可是/总感觉/不太对 | ★☆☆ |
| 强负 | 离谱！/凭什么/崩溃 | ★★★ |

**快速修正动作**：
- 动作A：在平铺直叙段末尾加1句情绪句
  - 例："数据显示增长了23%。" → "数据显示增长了23%。说实话，这个数字有点超出预期。"
- 动作B：在连续正极性中插入1个疑虑
  - 例："效果很好。大家都很满意。" → "效果很好。大家都很满意。不过，真的没有隐患吗？"

---

### 检查3：词汇温度（Vocabulary Temperature）

**检查目标**：确保近期文本的词汇温度有高低分布，不是"常温"聚集。

**检查方法**：
1. 扫描最近500字，提取所有实词
2. 按温度分类统计
3. 检查以下指标：

| 指标 | 通过标准 | 不通过处理 |
|------|----------|-----------|
| 高温词数量 | ≥1个 | 插入1个口语/情绪词 |
| 低温词数量 | ≥1个 | 插入1个专业/古雅词 |
| 常温词占比 | ≤65% | 替换部分常温词 |
| AI标记词 | 0个 | 立即替换 |

**AI标记词紧急替换表**：

| 检测到 | 替换为 |
|--------|--------|
| 值得注意的是 | 注意/留心/看到没 |
| 综上所述 | 总之/说到底/一句话 |
| 与此同时 | 同时/另一边/再说 |
| 需要指出的是 | 得说一句/讲真 |
| 在当今社会 | 现在/这个时代 |
| 众所周知 | 谁都知道/都知道 |
| 不可忽视 | 别小看/别当空气 |
| 毋庸置疑 | 没得跑/肯定 |

**快速修正动作**：
- 动作A：将1个常温词替换为高温词
  - 例："这个方法很好" → "这个方法绝了"
- 动作B：将1个常温词替换为低温词
  - 例："不同的情况" → "异质性的场景"

---

### 检查4：素材锚定（Material Anchoring）

**检查目标**：确保近期文本有足够的信息锚点，不是空洞论述。

**检查方法**：
1. 扫描最近500字，标记所有信息锚点
2. 锚点类型：人名+身份、具体数字+来源、时间日期、地点名称、事件名称
3. 检查以下指标：

| 指标 | 通过标准 | 不通过处理 |
|------|----------|-----------|
| 锚点总数 | ≥2个 | 补充1-2个具体信息 |
| 锚点类型 | ≥2种类型 | 补充不同类型的锚点 |
| 数据锚点 | ≥1个 | 补充具体数字 |
| 抽象表述 | ≤3个 | 将抽象表述具体化 |

**抽象→具体快速替换**：

| 抽象 | 具体化方案 |
|------|-----------|
| 很多人 | 具体数字或"10个人里有7个" |
| 最近 | "过去30天"/"今年以来" |
| 大幅 | 具体百分比 |
| 某公司 | 具体公司名 |
| 效果显著 | 具体指标变化 |
| 行业领先 | 具体排名或份额 |

**快速修正动作**：
- 动作A：将"很多人"替换为具体比例
  - 例："很多人都在用" → "用户量已经突破2000万"
- 动作B：在论述段补充1个数据
  - 例："成本降低了" → "成本从每月3万降到了8000"

---

### 检查5：句法变形（Syntactic Variation）

**检查目标**：确保近期文本的句式有变化，不是主谓宾的流水线。

**检查方法**：
1. 扫描最近500字，标注每句的句式类型
2. 句式类型分类：

| 句式类型 | 示例 | 说明 |
|----------|------|------|
| 主谓宾 | "他辞职了" | 基础句式 |
| 被动 | "方案被否了" | 被动语态 |
| 倒装 | "好家伙，这一步走错了" | 语气前置 |
| 设问 | "怎么办？只有一条路。" | 自问自答 |
| 排比 | "没有犹豫，没有退缩，没有回头" | 3个以上并列 |
| 省略 | "于是。" | 极简 |
| 插入 | "说实话，我不太确定" | 插入语 |
| 双重否定 | "不是不可能" | 否定表肯定 |

3. 检查以下指标：

| 指标 | 通过标准 | 不通过处理 |
|------|----------|-----------|
| 句式类型数 | ≥3种 | 增加句式变化 |
| 主谓宾占比 | ≤50% | 替换部分为其他句式 |
| 连续同句式 | ≤2句 | 第3句必须换句式 |
| 设问/反问 | ≥1个/500字 | 插入1个反问句 |

**快速修正动作**：
- 动作A：将2个连续主谓宾改为1个设问
  - 例："这个功能很重要。它能解决很多问题。" → "这个功能重要吗？重要到能解决80%的问题。"
- 动作B：将1个陈述句改为倒装
  - 例："这件事让我很意外" → "意外，真的意外。这件事让我始料未及。"

## 自检执行流程

```yaml
realtime_check:
  trigger: every_500_chars
  checks:
    - id: sentence_length
      name: "句长交替"
      priority: 1
      auto_fix: true
      max_fix_time: 3s

    - id: emotional_anchor
      name: "情绪锚定"
      priority: 2
      auto_fix: true
      max_fix_time: 3s

    - id: vocabulary_temp
      name: "词汇温度"
      priority: 3
      auto_fix: true
      max_fix_time: 3s

    - id: material_anchor
      name: "素材锚定"
      priority: 4
      auto_fix: false    # 需要WebSearch时不能自动修
      max_fix_time: 5s

    - id: syntactic_variation
      name: "句法变形"
      priority: 5
      auto_fix: true
      max_fix_time: 3s

  on_check_fail:
    log: true
    fix_immediately: true     # 可自动修的立即修
    fix_after_paragraph: true # 需要信息的段落写完后修
    max_accumulated_issues: 5 # 累积问题超5个则暂停写作先修
```
